																			
	迁徙区(城镇)设计																		
																			
	一	设计目的																	
																			
		1	引导玩家进行聚集，为社交提供基础载体。																
		2	迁移区在地图分为三种，逐渐递进，分别是小型落脚点、中立迁移区、GVG区域城镇；分别承载：迁移教学，散人玩家的聚集地，联盟社群中长周期的持续地缘争夺目标。																
		3	城镇提供复合的功能需求，包括更高的资源获取、功能的解锁、PVX的玩法、任务收发,。从规模最小到最大，功能会越来越完整。																
																			
	二	需求概述																	
																			
		1	临时落脚点																
			1.1	基础资源购买的商店															
			1.2	资源采集															
			1.3	功能解锁															
			1.4	任务NPC															
																			
		2	中立城镇																
			2.1	迁入城镇可以采集															
			2.2	城镇里面会有更丰富的剧情发放NPC															
			2.3	城镇内有一些公共的挑战玩法（挑战boss、探索秘境之类的）															
			2.4	城镇一些包含更多种类的商店（PVX、交易行、高级商店）															
			2.5	城镇包含PVX玩法区域															
																			
		3	GVG区域的城镇																
			3.1	承载攻防占领规则，额外包含攻防建筑和中心建筑、作为GvG目标玩法要素，会有更复杂的层级结构															
			3.2	有更高的集体产出收益和更好的商店道具出售															
			3.3	提供类似霸业积分的赛季判定数值（待定）															
																			
			7日版本不做GVG玩法，暂时不开发攻城玩法，只做城镇迁入、城镇矿区需求

	城镇布局结构																		
																			
	1、	城镇布局概述																	
		1.1	城镇均由2层大结构构成，分为中间的城镇建筑区+外围的迁城区																												
	2、	建筑区																	
		2.1	由中心建筑+附属建筑组成的建筑群																
		2.2	建筑暂时无功能																
		2.3	建筑会搭配NPC进行摆放，功能（任务/商店）交互会挂接在NPC上																
		2.4	城镇区由制作人员手工制作摆放组成，包括建筑、NPC																
																			
	3、	迁城区																	
		3.1	玩家可以在迁城区进行扎营，之后会把玩家归属为当前迁城区域的居民																
		3.2	对于小型聚落和中立区的城镇，不包含归属概念，所有玩家可自由迁入，非中立区城镇有归属，占领之后，公会的玩家才能迁入																
		3.3	在城镇范围内，仅有迁城区支持营地展开，其他区域不支持营地展开																
		3.4	迁入时，是按照迁城车位的逻辑来的，每个车位对应一个玩家营地																
		3.5	迁城区由策划通过编辑器配置出来																
			3.4.1	迁城区与的逻辑应该是单个迁城位置为N*N的格子															
			3.4.2	一个整个的连贯迁城区域，由M*M个3.4中的格子拼接而成，策划只需要输入M的参数，之后在手动删减和调整格子															
																			
	4、	城墙+攻防建筑（待定，GVG城镇再做）																	
																			
	迁城流程																		
																			
	1、	如何迁城																	
		1.1	玩家驾车进入城市范围，会提示如果允许迁入（归属判断），会有选择车位迁城的流程																
		1.2	进入流程后，会打开一个迁城区域的俯视视图，上面显示当前城镇的车位分布以及迁入状态、已经迁入车位上的玩家信息（名字、公会等）																
		1.3	玩家可以选择空位进行迁入，确定后进入迁移流程（因为是公共迁城区，交互要考虑其他玩家操作状态的公共表现设计）																
																			
	2、	迁移流程																	
		2.1	玩家选择空位并开启迁移，会播放一个过程动画（玩家车开进城、开到车位里面）																
		2.2	动画播完之后，回到场景之后，玩家的车就已经移动到车位内，并且播放展开的效果。之前会提示【迁入成功】																
																			
	3、	迁走流程																	
		3.1	玩家在车位内收起营地，会触发是否迁走的提示，点击确认会播放迁走的动画（营地收起来，车开出车位，开出城门口）																
		3.2	动画完成后，玩家的车已经移动到城镇门口，此时车保持收起来的状态																
																			
	城镇矿区																		
																			
	1、	概述																	
		1.1	城镇内设有矿区，迁入城镇的居民可以开采城镇里面的矿区																
		1.2	矿区是一个永久性的资源（没有总量，不会消失）																
																			
	2、	矿区开采																	
		2.1	迁入城镇后，可以派遣比宝前往矿区进行资源开采																
		2.2	具体操作为选择城镇矿区-->选择比宝-->派遣，需要选中矿区（物件）之后派遣（交互逻辑和建筑派遣是一样的）																
																			
	3、	资源获取逻辑																	
		3.1	矿区开采的逻辑为按照当前的资源获取效率，稳定的获取累积的资源																
		3.2	矿区不设资源存量的概念，不会消失，是一个固定的永久性资源																
																			
	4、	资源获取数值																	
		4.1	基础效率：	矿区本身自带的属性，不同等级的矿区，提供的基础资源获取效率不同															
		4.2	效率加成：	其他系统提供的额外附带效率，例如：比宝的效率加成值、比宝派遣数量不同一个的加成倍率、同盟加成等等（这个各自的功能单独自己提需求）															
																			
	5、	矿区参数																	
		5.1	矿区等级：	等级参数															
		5.2	产出效率：	开采过程中单位时间的资源产出数量															
		5.3	矿区类型：	类型参数，会关联产出的基础资源类型															
		5.4	开采上限：	同一时间允许开采的玩家数量上限															
		5.5	派遣上限：	一个矿区单个玩家可以派遣的比宝上限															
		5.6	开采时限：	玩家每天可以在矿区内开采的时间上限															
																			
	6、	全局参数																	
		6.1	单个城镇玩家开采矿区的总数量上限																
		6.2	单个城镇每种类型矿区玩家开采的数量上限																
		6.3	每天在城镇内采集的总时间上限																
																			
	7、	矿区表现																	
		7.1	需要有比宝在矿区开采的场景内表现																
		7.2	根据性能显示合理的比宝数量，优先显示自己的、其次好友的、其次盟友、最后其他人的																
																			
	8、	离开城镇																	
		8.1	玩家不再作为城镇居民之后（迁走、被打飞等），打工的比宝会收回，同时邮件通知玩家
