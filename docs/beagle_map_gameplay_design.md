## 1. 项目背景 (Context)

基于 Beagle 项目的核心架构，我们明确以下与大地图玩法紧密相关的基础元素：

*   **实体主城与移动基地:** 玩家拥有在大地图上可视、可交互且可移动的实体基地。
*   **全图不分线 (No Sharding):** 大地图所有区域（包括PVP/GVE/安全区）均采用单一大世界架构，不存在分线机制。
*   **单一维度社交:** 所有的社交行为（包括轻社交和强对抗）均发生在同一物理层面上，无需处理跨线交互问题，所有玩家在地理位置上是真实共存的。
*   **城镇作为核心节点:** 城镇不仅是功能点，更是GVE/GVG玩法发生的主要场所。

## 2. 设计目的 (Design Goals)

为了明确本次 PVP 地块的设计定位，我们将各类玩法的核心目的与体验进行了对比梳理：

| 玩法类型 | 核心目的 (Purpose) | 玩家体验 (Experience) | 备注 |
| :--- | :--- | :--- | :--- |
| **PVE** (探索与生存) | 满足基础生存需求，提供安全资源产出（采集/打怪），作为新手引导和日常成长基石。 | 轻松、无压力、个人成长感。 | 基础温饱 |
| **GVE** (城镇与BOSS) | 建立初步社交链接，多人协作挑战高难度 AI，产出进阶资源。 | 合作共赢、职业互补、团队成就感。 | PVE 进阶 |
| **GVG** (联盟战争) | 承载大型联盟领土争夺，决定服务器政治格局和核心资源分配。 | 宏大史诗感、集体荣耀、纪律与执行。 | 顶级社交 |
| **PVP** (对抗与博弈) | **验证个人实力？ 投放稀缺资源？ 联盟社交的前置渐进？小队社交的建立？日常的宣泄出口？** | **自我证明？ 惊喜收获？ 社交羁绊？ 战术配合？ 情绪释放？** | **本案重点，向 GVG 渐进** |

## 3. 参考玩法分析 (Benchmark Analysis)

在设计具体方案前，我们深入拆解了业界成熟的 PVP 玩法模式。在 PVP 地块设计上，我们**明确排除 GVG (大型公会战)**，对抗规模被锁定在 **单人 (Solo)** 到 **小队 (Squad)** 的范畴内。因此，我们通过 **风险回报单轴模型 (Risk-Reward Axis)** 来对参考游戏进行落位分析，以便确认 Beagle 游戏的落点。

### 3.1 风险回报轴 (Risk-Reward Axis)

由于本方案明确排除了 **GVG (大型联盟战)**，原本的“规模-风险”双轴模型可简化为 **“风险-回报”单轴模型**。

我们根据**死亡惩罚**和**收益预期**的程度，将参考玩法进行了如下映射：

<!-- SPECTRUM_VISUALIZATION_BENCHMARK -->

*   **Zone 3 (Low Risk):** **Benchmark 1** (WoW/Albion Yellow)。主打无损娱乐，作为泛用户入口。
*   **Zone 2 (Mid Risk):** **Benchmark 2** (PUBG/Division)。主打局部冲突与爆发，作为日常调味。
*   **Zone 1 (High Risk):** **Benchmark 3** (Tarkov/Albion Black)。主打全额掉落与博弈，作为核心追求。

### 3.2 深度拆解

<div class="benchmark-card">

#### **Benchmark 1: Albion Online (Yellow Zone) / WoW (Battlegrounds) (低风险竞技)**

*   **PVP 地块目的:** 降低 PVP 门槛，提供“安全出口”，让泛用户体验对抗乐趣。
*   **核心体验:** **“爽快与发泄” (Sport & Fun)**。享受纯粹战斗，无心理负担。

| **社交规则** | **阵营免伤 / 鼓励抱团**。人多即优势，积分制引导团战。 |
| **奖励投放** | **中等 (Medium)**。荣誉点数/外观/成就，不破坏核心数值。 |

## 4. 推荐方案 (Proposed Concepts)

基于上述分析，结合 Beagle 的轻量化社交属性，我们构思了三个层层递进的 PVP 地块方案 (CP: Concept Proposal)。

### CP1: 自由搏击俱乐部 (Fight Club)

<div class="benchmark-card">

#### **核心概述**

*   **玩法流程:** 地图固定区域开启乱斗 -> 联盟/小队/散人进场 -> 击杀敌对玩家获得积分 -> 结束后按个人积分结算奖励。
*   **一句话体验:** “我要打十个！” —— 享受纯粹的乱斗快感，无死亡掉落心理负担，联盟只是你的后盾。

<!-- CP_VISUAL_1 -->

| 维度 | 详情 |
| :--- | :--- |
| **设计目的** | **低风险低回报 (Low Risk)**。为联盟提供练兵场，培养团队配合；为散人提供无负担的乱斗机会。 |
| **参与单位** | **联盟 (Alliance) + 散人 (Freelancer)**。允许多家联盟同时入场混战，散人亦可自由乱入。 |
| **社交规则** | **联盟免伤 (Alliance Safe)**。**同联盟玩家免伤**。散人无阵营保护。战场内可能是 3-5 个联盟 + 大量散人的大混战。 |
| **开启周期** | **固定时段 (Scheduled)**。如每日晚间 20:00 - 21:00 开启，集中在线活跃。 |
| **奖励与重要性** | **个人积分与荣誉**。**完全按个人表现（伤害/击杀）结算**。产出【个人荣誉点】、【限定皮肤】。不投放联盟资金，纯粹的个人实力秀场。 |

</div>

### CP2: 幸运流星 - 趣味乱斗 (Meteor Carnival)

<div class="benchmark-card">

#### **核心概述**

*   **玩法流程:** 随机坠落彩色流星 -> 玩家需**读条占领 30秒**（受击打断） -> 获得【趣味礼盒】。
*   **一句话体验:** “不是为了变强，只是为了快乐。” —— 全服唯一的发光头套，谁不想要呢？

<!-- CP_VISUAL_2 -->

| 维度 | 详情 |
| :--- | :--- |
| **设计目的** | **纯娱乐 (Pure Fun)**。提供大地图上的“社交谈资”和无压力的混乱乐趣。绝不产出数值成长道具，避免逼肝。 |
| **参与单位** | **单人 (Solo) / 乐子人**。无论战力高低，只要运气好或手速快，人人都有机会。 |
| **社交规则** | **无视盟友 (Free for All)**。**联盟关系在此失效**，无法豁免伤害。鼓励“六亲不认”的抢夺乐趣，即使是联盟盟主也可能被成员抢走头套。 |
| **开启周期** | **高频 (High Frequency)**。每 30-60 分钟随机刷新，全服可见。 |
| **奖励与重要性** | **限时外观/玩具**。产出【7天限时发光称号】、【变身药水】（变身Boss模型）、【全服烟花】。主打炫耀和整活。 |

</div>

### CP3: 遗物猎杀 - 绝境撤离 (Relic Hunt)

<div class="benchmark-card">

#### **核心概述**

*   **玩法流程:** PVP 地块刷新【宝箱】与【守护宝可梦】 -> 玩家需先击败强力 Boss -> 拾取宝箱（此时无法立即开启或带出） -> 在 PVP 地块任意区域发射求救信号 -> **坚守 5 分钟** -> 运输机抵达，携带宝箱撤离。
*   **一句话体验:** “从猎人变成猎物。” —— 击败 Boss 的瞬间，才是真正噩梦的开始。

<!-- CP_VISUAL_3 -->

| 维度 | 详情 |
| :--- | :--- |
| **设计目的** | **高风险高回报 (High Risk)**。提供顶级 PVE 挑战与极致的 PVP 追逐体验，验证小队攻坚与生存能力。 |
| **参与单位** | **精英小队 (Elite Squad)**。推荐 3-4 人配置，需兼顾输出（打 Boss）与控制/防守（打玩家）。 |
| **社交规则** | **小队唯一 (Squad Only)**。仅小队成员（3-4人）免伤。**同联盟但不同小队**的成员，视为中立单位（可攻击），打破联盟人海战术，强制回归小队配合。 |
| **开启周期** | **中频 (Mid Frequency)**。每 2-4 小时随机刷新，全服公告。 |
| **奖励与重要性** | **顶级稀缺 (Exclusive)**。产出独占的【传说级装备图纸】或【神兽进化石】。不仅是数值提升，更是服务器内的荣耀象征。 |

</div>

## 5. 对比与总结 (Comparison & Summary)

<!-- SPECTRUM_VISUALIZATION_CP -->

| 维度 | CP1: 自由搏击俱乐部 (Fight Club) | CP2: 幸运流星 (Carnival) | CP3: 遗物猎杀 (Relic Hunt) |
| :--- | :--- | :--- | :--- |
| **核心体验** | **娱乐与发泄** (Fun/Sport) | **欢乐与整活** (Party/Fun) | **生存与贪婪** (High Stakes) |
| **死亡惩罚** | **无掉落 (耐久损耗)** | 部分掉落 (黑货) | **全额掉落 (Full Loot)** |
| **主要奖励** | **个人积分/外观荣誉** | **限时外观/变身道具** | 顶级装备/核心资源 |
| **社交规则** | **联盟免伤/多方混战** | 混乱乱斗/无视公会 | 强制小队/无视公会 |
| **参与单位** | **联盟 (Alliance) + 散人** | 投机单人 (Solo) | 精英小队 (Squad) |
| **Beagle结合** | 道馆模拟战/联盟大会 | 变身大岩蛇/烟花庆典 | 精灵濒死/道具掉落 |

### 推荐结论与演进路线 (Roadmap)

建议采用 **“金字塔式” (Pyramid Structure)** 的 PVP 架构，根据开发难度与用户覆盖度进行分阶段落地：

<!-- PYRAMID_VISUALIZATION -->

#### **核心策略总结**

*   **CP1 (基础/7日目标)**: **风险最低，受众最广**。作为“常驻安全出口”，它是泛用户接触 PVP 的第一站。当前版本应全力实现此玩法。
*   **CP2 (聚集/调味剂)**: **随机事件，活跃气氛**。作为常驻玩法的补充，利用“流星”等随机机制制造局部热点，打破常规节奏。
*   **CP3 (重度/未来)**: **强系统，高回报**。涉及复杂的 Boss AI、撤离逻辑与掉落机制，是长线运营的深度内容，建议后续开发。
